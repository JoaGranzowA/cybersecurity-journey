### 1. Cloud computing fundamentals

#### What is cloud computing?

Cloud computing is on-demand delivery of computing services, instead of having on-premise servers, you rent these resources like storage, databases, and software.

#### Shared responsibility model

Shared responsibility model works in the way that the client and the cloud service provider share responsibility, it defines which security tasks are handled by the cloud provider and which are handled by the client.

#### Cloud deployment models: public, private, hybrid, multi-cloud

**Public Cloud:** In public cloud services are delivered over the public internet and shared over multiple organizations.

**Private Cloud:** A private cloud is used exclusively by a single organization. It can be hosted either on-premises or by a third-party provider, but its resources are not shared with others.

**Hybrid Cloud:** Hybrid cloud uses both public and private, usually they store the sensitive data on-premise and the other type of data in the public cloud.

**Multi-Cloud:** The use of 2 or more different public cloud providers.

#### Cloud service types: IaaS, PaaS, SaaS

**IaaS:** In infrastructure as a service you rent the resources, like the virtual servers, the storage, and networks. You are responsible for installing and managing the OS and the tools.

**PaaS:** In platform as a service you pay for a platform for developers to build, test, and deploy apps without worrying about server management.

**SaaS:** In software as a service you pay for the final product, the app is ready to use. You simply log in and use it.

#### CapEx vs OpEx

**CapEx:** Capital expenditure means that you spent money building infrastructure, maintain, or improve before you can use it, CapEx requires large upfront investment + long depreciation.

**OpEx:** In operational expenditure you pay as you use the resources, you don't have to maintain or improve the datacenters. You can stop, start, or scale the resources based on demand, OpEx spreads cost over time.

#### Consumption-based pricing

Consumption-based pricing is applied on OpEx, you pay as you go, no upfront commitment, and you can stop, scale up or scale down services based on demand.

### 2. Azure core architecture

#### Regions, availability zones, region pairs

A region is a geographic location, where a provider has multiple data centers. An availability zone is a physically separate datacenter within an Azure region, each with independent power, cooling, and networking. Availability zones exist so that if one zone fails, others continue operating.

A region pair consist of 2 regions within the same geography, these help to save data in case of a disaster recovery. Region pairs are usually at least 300 miles apart and Azure platform updates are rolled out sequentially to minimize risk of simultaneous failure.

#### Resources, resource groups, subscriptions

**Resources:** Are instances of a service that we can create and manage.

**Resource groups:** Logical containers used to group and manage resources.

**Subscriptions:** Billing and access boundaries that contain resource groups and resources. They help separate environments, projects, departments, or cost centers.

#### High availability, scalability, reliability, predictability

**High availability:** The ability of a system to remain accessible and operational with minimal downtime, even during failures.

**Scalability:** The ability of a system to handle increased demand by adding or adjusting resources.

**Reliability:** The ability of a system to consistently perform its intended function without failure over time.

**Predictability:** The ability to ensure consistent performance and cost, so the system behaves as expected under different conditions.

#### Vertical vs horizontal scaling

**Vertical scaling:** Increasing the capacity of a single resource by adding more power.

**Horizontal scaling:** Increasing capacity by adding more instances of a resource and distributing the load.

#### Azure AD/Entra ID

Entra ID is a cloud-based identity and access management service that manages users and groups, authenticates sign-ins, and control access to resources using roles.
Azure subscriptions trust only one Entra ID tenant.

#### Management Groups

Management groups help to apply RBAC, policies, budget, and manage multiple subscriptions from a single point.
Management groups have a hierarchical structure, where configurations are inherited to the child management groups, subscriptions, and resources. A subscription can belong to only one management group at a time.

#### Azure Resource Manager (ARM)

ARM is the deployment and management service for Azure, it creates, updates, and deletes resources. ARM not only does that, it also applies authorization and policies. It deploys resources using templates (JSON/Bicep). One key thing to keep in mind is that all tools interact with ARM, so you cannot bypass it.

ARM also checks RBAC permissions when a user tries to manage Azure resources.

#### Azure Arc

Azure Arc extends management and governance to resources outside Azure, which means we can manage on-premises servers, virtual machines in other clouds, Kubernetes clusters, etc.
With Azure Arc, we can apply policies, RBAC, security features, and more to those resources.

#### ARM Templates

Declarative JSON files that describe Azure resources and their configuration. They are deployed through tools like Azure CLI, PowerShell, the Portal, or CI/CD pipelines. Once submitted to ARM, it compares the declared state with the actual state in Azure and adjusts whatever is needed.

Used for Infrastructure as Code: the same template can be deployed multiple times producing the same result, which makes infrastructure repeatable, versionable in Git, and consistent across environments.

### 3. Compute and containers

#### Resources Required for Virtual Machines

A virtual machine is not a single resource; it depends on multiple components, including compute, storage, networking, and security.
Choosing the right VM size (CPU and memory) is important for performance and cost.

#### Virtual Machine Scale Sets

Virtual Machine Scale Sets (VMSS) allow you to deploy and manage multiple identical VMs and scale them automatically based on demand.

#### Azure Container Instances

Azure Container Instances (ACI) allow you to run containers without managing virtual machines, providing a simple and fast way to deploy applications. You define CPU and memory at deployment time.

#### Azure Kubernetes Service

Azure Kubernetes Service (AKS) is a managed Kubernetes service that orchestrates and scales containerized applications across multiple nodes.

#### Azure App Service

Azure App Service is a PaaS offering that allows you to build and host web applications without managing infrastructure.

#### Azure Serverless

Azure Serverless allows running code without managing infrastructure using an event-driven model. Azure Functions executes code in response to events, while Logic Apps automate workflows and integrations.

#### Azure Virtual Desktop

Azure Virtual Desktop (AVD) allows users to access virtual desktops and applications hosted in Azure from any device.

#### Azure Dedicated Hosts

A physical server dedicated to a single customer in Azure. We create Host Groups, which contain one or more Dedicated Hosts. Our VMs run inside those hosts.
Used when there are compliance requirements that forbid shared hardware, when leveraging BYOL licenses that are priced per physical core, or when the customer needs to control maintenance windows.

### 4. Networking

#### Azure Virtual Networks

Azure virtual networks are used to create private networks in Azure. A VNet is limited to a single region and cannot span across regions.
VNets use private IP address ranges defined by RFC1918.

#### Virtual Networks Subnets

Subnets are created within a VNet by dividing the IP address range into smaller segments, so each subnet must have a defined IP range.

#### VNet Peering

VNet peering allows communication between VNets using private IP addresses.
When using multiple VNets, their IP address ranges must not overlap.

#### Site-to-Site VPN

Site-to-Site VPN is used to connect an on-premises network to an Azure virtual network over the internet. It creates an encrypted tunnel between the networks and allows them to communicate using private IPs.
It requires a VPN gateway in Azure and a compatible VPN device on-premises.
Site-to-Site VPNs can be configured as route-based or policy-based.

#### Policy-based VPNs

Policy-based VPNs use policies to define which traffic is encrypted and sent through the tunnel. They are more limited and typically used for compatibility with older VPN devices.

#### Route-based VPN

Route-based VPNs use routing tables to direct traffic through the VPN tunnel and support dynamic routing. They are more flexible and recommended for most scenarios.
Route-based VPNs support both Site-to-Site (S2S) and Point-to-Site (P2S) connections.

#### Point-to-Site

Point-to-Site (P2S) allows individual clients (VMs, laptops, etc) to connect securely to an Azure Virtual Network.

#### Active-Passive configuration

Active-Passive (A-P) configuration uses one active VPN gateway and one standby instance for failover.

#### Active-Active configuration

Active-Active (A-A) configuration uses two active VPN gateway instances for higher availability and better performance.

#### Azure ExpressRoute

ExpressRoute provides a private and dedicated connection between on-premises infrastructure and Azure without using the public internet.
Traffic flows through a carrier over a private network to Microsoft routers. It offers high reliability, lower latency, and more consistent performance compared to VPN connections.

#### Private Peering

Private Peering is used to connect on-premises networks to Azure Virtual Networks using private IPs. It uses ExpressRoute to keep the connection private.

#### Microsoft Peering

Microsoft Peering is used to access Microsoft public services such as Microsoft 365 and Azure PaaS services. It uses public IP addresses, but traffic still travels over the private ExpressRoute connection.

#### Service Endpoints

Service endpoints allow Virtual Networks to connect to Azure services using the Azure backbone network. They extend the VNet identity to Azure services such as Azure Storage and Azure SQL Database. 
Service Endpoints use public IP addresses of the Azure service but restrict access to specific VNets and subnets.

#### Private Endpoints

Private Endpoints provide a private connection from a Virtual Network to Azure services using a private IP address. They assign a private IP from the VNet to the Azure service, making it accessible as if it were inside the VNet. Also we can connect on-premises networks through a VPN or ExpressRoute.

#### Public Endpoints

It is an address that we can communicate to from the internet (Public) or from services in a virtual network. It is internet routable, I can get to it from anywhere, but I still need to authenticate depending on the service.
We could use a service level firewall to allow communication only from specific public IPs or VNets. It is the default for most PaaS services.

### 5. Storage and databases

#### Storage Account

A storage account is a service used to store different types of data, such as blobs, tables, queues, and files.
Storage accounts have two performance options: Standard, and Premium.
Standard performance uses GPv2 (General Purpose v2).
On the other side, Premium performance has three options, Block blob, Page blob, and File Shares.

#### Locally Redundant Storage

In LRS there are three copies of the data, but they are in the same datacenter within the region. If something happens to that datacenter, our data may be not available.

#### Zone Redundant Storage

ZRS also keeps three copies of the data, but they are stored in different zones. In this case we still have access to the data if one availability zone goes down.

#### Geo-Redundant Storage

GRS keeps six copies of the data: three in the primary region and three in a secondary region, which is the paired region. This protects against a full regional failure.
The replication to the secondary region is asynchronous, so if the primary region goes down suddenly, we could lose the last few seconds of writes that were not yet replicated.

#### Geo-Zone-Redundant Storage

GZRS is a combination of GRS and ZRS. It keeps six copies of the data. There are three copies in the primary region, but in different availability zones, and the other three are stored in a secondary region.

#### Standard vs Premium

Standard has LRS, ZRS, GRS, and GZRS.
Premium has only LRS and ZRS.
This means Premium does not support geo-redundancy, so if a workload needs both high performance and protection against regional failures, the redundancy must be handled at the application level.

#### GRS and GZRS Read Access

The Read Access option allows us to get a second endpoint in the secondary region to read the data.

#### Binary Large Object (Blob)

A Blob is basically any type of data we want to store, such as images, videos, documents, backups, logs, or files.

#### Block Blob

A Block Blob is used to store files such as images, videos, documents, backups, and logs. It stores the data in blocks and is the most common type of Blob.
Block blobs use a flat namespace, which means there is no real folder hierarchy.

#### ADLS Gen2

ADLS Gen2 is built on top of Blob Storage and is used to store large amounts of data for analytics and big data workloads. It also supports a hierarchical namespace.

#### Page Blob

A Page Blob is used to store virtual hard disks for Azure Virtual Machines. It is optimized for frequent read and write operations.

#### Append Blob

An Append Blob is used to append data to the end of a blob. It is commonly used for logs.

#### Access Tiers

Access tiers define how frequently the data is accessed. Hot is for frequently accessed data, Cool is for less frequently accessed data, Cold is for rarely accessed data, and Archive is for long-term storage.

#### Standard HDD

Standard HDD is the cheapest disk option. It uses traditional hard disk drives and is used for workloads that do not need high performance.
It is commonly used for backups, test environments, or infrequently accessed workloads.

#### Standard SSD

Standard SSD provides better performance than Standard HDD and is used for light workloads, web servers, dev/test environments, and small applications.
It also supports burst performance, which allows the disk to temporarily increase its performance when there are short peaks of activity.

#### Premium SSD

Premium SSD is used for production workloads that need high performance and low latency. It supports burst performance, which is useful when the workload has temporary peaks of activity. It also supports performance tiers, which allow us to increase the disk performance without increasing the disk size.

#### Premium SSD v2

Premium SSD v2 provides higher performance and more flexibility than Premium SSD. It allows us to configure the disk size, IOPS, and throughput independently.
Premium SSD v2 is dynamic because we can adjust IOPS and throughput depending on the workload.

#### Ultra Disk

Ultra Disk is the highest performance disk option in Azure. It is used for very demanding workloads, such as large databases, SAP HANA, and applications that require very high performance. It allows us to configure IOPS and throughput independently.
Ultra Disk is dynamic because we can change its performance settings without restarting the VM.

#### Azure Files

Azure Files is a fully managed file share service in Azure. It allows us to create shared folders in the cloud that can be accessed by multiple users, applications, or virtual machines. Azure Files is accessible using the SMB protocol, just like a regular file share on a local network.

#### Azure File Sync

Azure File Sync is a service that allows us to synchronize an on-premises file server with an Azure file share.
It lets users keep using a local file server, while Azure keeps a synchronized copy in the cloud.

#### Sync Group

A Sync Group defines the synchronization relationship between an Azure file share and one or more on-premises Windows Servers. It contains a cloud endpoint and one or more server endpoints. The cloud endpoint is the Azure file share, and the server endpoint is the folder in the on-premises Windows Server.

#### Cloud Tiering

Cloud Tiering is a feature of Azure File Sync that helps save space on the on-premises server. With Cloud Tiering, frequently used files are kept locally, while less frequently used files are stored in the Azure file share. The files still appear on the local server, but some of them are only downloaded when users open them.

#### Queue Storage

Queue Storage is used to store messages that can be processed later. It works with FIFO, which means First In, First Out. This means that the first message added to the queue is usually the first message to be processed.
Queue Storage is commonly used to communicate between different parts of an application.

#### Table Storage

Table Storage is used to store structured NoSQL data. It stores data as key-value pairs, which means each item has a key that identifies it and a value that contains the information.
It is useful for simple and fast data storage without using a relational SQL database.

#### Static Website

Static Website allows us to host static web content directly from a storage account.

#### Azure Storage Explorer

Azure Storage Explorer is a free desktop tool used to manage Azure Storage resources. It allows us to upload, download, view, and manage data in storage accounts, such as blobs, Azure file shares, queues, and tables.

#### AzCopy

AzCopy is a command-line tool used to transfer data in Azure Storage. It can copy data from a local computer to Azure Storage, from Azure Storage to a local computer, between storage accounts, and between supported cloud providers. It is useful when we need to move large amounts of data efficiently.

#### Azure Migrate

Azure Migrate is a service used to discover, assess, and migrate on-premises workloads to Azure. It helps migrate servers, databases, web applications, and virtual desktops to Azure.

#### Azure Data Box

Azure Data Box is used to move large amounts of data to Azure using a physical device instead of transferring everything over the internet.
Microsoft sends the device to the customer, the customer copies the data into it, and then sends it back to Microsoft so the data can be uploaded to Azure.

#### Azure Data Box Options

Data Box Disk is used for smaller data transfers using SSD disks. It supports up to 5 disks, with 8 TB each, giving around 35 TB of usable capacity.
Data Box is used for large data transfers using a physical device. The traditional Data Box device has around 80 TB of usable capacity. Newer Data Box options can support 120 TB or 525 TB, depending on the device.
Data Box Heavy is used for very large data transfers. It has around 770 TB of usable capacity.

#### SQL Server

SQL Server is Microsoft's relational database solution. In Azure, it can be implemented as a PaaS service.

#### Azure SQL DB

Azure SQL Database is a PaaS service that helps us implement a relational database. It has some restrictions compared to on-premises SQL Server, so not all SQL Server features are fully supported.

#### Azure SQL MI

Azure SQL Managed Instance is also a PaaS service in Azure, but the difference with Azure SQL DB is that it provides high compatibility with on-premises SQL Server.

#### Managed Databases Solutions

There are many Open-Source SQL options in Azure like MySQL, PostgreSQL, MariaDB. These are great options if you are migrating one of those databases.
Also PostgreSQL in particular, has an extension called citus, which allows us to have a multi-server architecture, where the data is sharded over multiple servers.

#### Hyperscale

Hyperscale is a service tier for Azure SQL Database. It is designed for very large databases that need high scalability, fast backups, and fast restore times.

#### Azure Cosmos DB

Azure Cosmos DB is a fully managed NoSQL database service in Azure. It is designed for applications that need high scalability, low latency, and global distribution.
Cosmos DB can store different types of NoSQL data, such as documents, key-value data, graphs, and column-family data.

#### Azure Marketplace

Azure Marketplace is an online store where we can find, buy, and deploy applications, services, and solutions from both Microsoft and third-party providers. The resources are pre-configured and ready to deploy in our Azure environment.

### 6. Analytics, AI, and IoT

#### IoT

IoT or Internet of Things refers to a vast network of devices that are embedded with sensors and software. This allows these devices to connect and communicate with each other over the internet.

#### Azure IoT Hub

Azure IoT Hub is a service that acts as a central message hub for bi-directional communication between the devices and the cloud.

#### Device Twin

Device Twin is a JSON document in the cloud that acts as a virtual representation of a device. It persists even when the device is offline, which lets us query the device state and apply configuration changes that get synced once the device reconnects.

#### Azure IoT Central

Azure IoT Central uses IoT Hub under the hood, but it is a SaaS solution. It provides dashboards, device templates, and rules out of the box, so we don't need to build our own app. It is the right choice when we want to deploy fast without development effort, sacrificing flexibility.

#### Azure Sphere

Azure Sphere is a secure end-to-end IoT solution for microcontroller-based devices. It combines certified hardware, a Linux-based operating system, and a cloud security service that handles updates and monitoring.

#### Azure Data Factory

A PaaS service for orchestrating ETL pipelines. It moves, transforms and loads data between many sources and destinations. Used when we need to consolidate data from multiple systems into a single place.

#### HDInsight

A managed service for running open-source big data frameworks in Azure (Hadoop, Spark, Kafka, HBase, Hive, Storm). Microsoft manages the infrastructure, we use the tools we already know from the Hadoop ecosystem. Useful when a company already works with these technologies and wants to move them to the cloud.

#### Databricks

A SaaS platform optimized for Apache Spark, with a collaborative notebook-based interface for data scientists and data engineers. Designed for big data processing, machine learning, and advanced analytics.

#### Azure Synapse Analytics

A unified analytics platform that combines data warehousing (SQL), big data processing (Spark), and data integration (pipelines) in a single service. It is designed for analyzing large volumes of data (TB to PB) using massively parallel processing (MPP). Used for building enterprise data warehouses and BI solutions, not for transactional workloads.

#### When to use which (Big Data services)

- **Data Factory** → orchestrate movement and transformation of data between systems.
- **HDInsight** → run open-source big data frameworks (Hadoop, Spark, Kafka) managed.
- **Databricks** → Spark-based platform for data science, ML, and collaboration.
- **Synapse Analytics** → unified platform: data warehouse + big data + pipelines + BI.

#### Azure Machine Learning

Azure's platform for building, training, and deploying ML models in the cloud.
Makes ML models available via API to endpoint, so the apps can talk to the endpoint.

#### Azure Cognitive Services (Azure AI Services)

Pre-built ML models. Some of the uses are language, speech, vision, and decision.

#### Azure Bot Service

Platform to build, train, and deploy virtual agents. Allows users to interact with through text or voice.

#### Azure DevOps

Platform that provides end-to-end toolchain for developing and deploying software.

#### Kanban Boards

Visual project management tool used to track work as it moves through various stages.

#### Azure Boards

Service that implements Kanban Boards in Azure.

#### Azure Pipelines

Service for compiling, testing, and implementing code with CI/CD.

#### Azure Artifacts

A service within Azure DevOps for hosting and managing packages. It allows teams to share private packages across projects, cache public packages, and control which versions are used in the organization.

#### GitHub Actions

GitHub's automation engine. Covers CI/CD plus other repository automation.

#### Azure Dev Test Labs

Service that provides us with an environment to build and test versions of our software.

### 7. Management and monitoring

#### Azure Portal

Web-based graphical interface. Best for exploring, learning, and viewing resources. Not ideal for repetitive tasks or large-scale deployments. Accessible from any device with a modern browser, including mobile.

#### Azure Mobile App

Limited mobile version of Azure Portal. Good for receiving alerts or performing basic actions.

#### Azure PowerShell

Cross-platform CLI based on PowerShell. Imperative, designed for automation. Common choice for teams with Windows background.

#### Azure CLI

Cross-platform CLI based on bash. Same purpose as Azure PowerShell, but better option for teams with experience in bash.

#### Cloud Shell

Browser-based terminal that comes with Azure PowerShell and Azure CLI pre-installed. Accessible via web browser, not from a local command line.

#### Azure Advisor

Service that provides recommendations about costs, security, reliability, performance, and operational excellence. Accessible through Azure Portal or API. Use Advisor when you want to receive alerts on **recommendations** (cost optimization, security improvements, etc).

#### Azure Monitor

The umbrella service for collecting, storing, and analyzing telemetry from Azure resources, subscriptions, and Entra ID. It supports metrics and logs. Use Monitor for **real-time metrics, logs, and alerts on resource behavior** (CPU usage, latency, etc).

#### Activity Log

Subscription-level log that records all control-plane operations. Retention: 90 days by default.

#### Metrics vs Logs

- **Metrics:** numeric data points sampled at regular intervals (CPU %, latency, IOPS). Lightweight, used for dashboards and real-time alerts.
- **Logs:** detailed event records with full context. More expensive but richer for investigation.

#### Diagnostic Settings

Per-resource configuration that defines where the resource's metrics and logs are sent. Possible destinations: Azure Monitor, Azure Storage, Event Hub, or Log Analytics Workspace.

#### Log Analytics Workspace (LAW)

The primary destination for logs intended for active analysis. Logs stored here can be queried using KQL. Default retention is 30 days, extendable up to 2 years. It is also the foundation for Microsoft Sentinel.

#### Event Hub

Used to stream Azure logs to external systems, typically third-party SIEMs.

#### Azure Service Health

Free Azure service that provides a personalized view of services status. Notifies us about service issues, planned maintenance, health advisories, and security advisories.
Also allows us to configure health alerts, and review the health history.

Use Service Health for:
- **Outages and incidents** affecting Azure services
- **Root Cause Analysis (RCA)** reports after an outage
- **Health advisories** — proactive notifications about service retirements and breaking changes
- **Planned maintenance** notifications

#### Monitoring services — when to use which

- **Azure Monitor** → metrics, logs, real-time alerts on resource behavior
- **Azure Service Health** → outages, RCAs, health advisories, service retirements
- **Azure Advisor** → recommendations (cost, security, reliability, performance)
- **Azure Cost Management** → spending analysis and control

### 8. Identity and security

#### Azure Defender for Cloud

A unified security service that combines security posture management (CSPM) and threat protection (CWP) for resources across Azure, on other clouds, and on-premises (through Azure Arc).

- **CSPM:** continuous assessments of security configuration. Provides secure score, recommendations, and compliance dashboards.
- **CWP:** real-time threat detections through plans.

#### Key Vault

Service for securely storing secrets, passwords, and certificates.
Can be controlled with two models:

- **Access Policies:** Legacy model, you grant permissions to every secret, key or certificate.
- **RBAC:** Uses predefined Azure user roles, allows us to grant permissions individually to each secret, key or certificate.

#### Managed Identity

An identity in Entra ID assigned to a resource so it can authenticate to other Azure services without credentials in code. Works like a user in RBAC — you can assign it roles the same way.

#### Microsoft Sentinel

Microsoft's SIEM/SOAR solution. It detects threats, performs investigations, and automates responses. Adds ML to give insight of what's happening in the environment. It contains other tools like hunting, notebooks, incidents, workbooks, and more.

#### Defense in Depth

A layered security strategy where each layer adds a barrier, so a single failure does not compromise the system. Microsoft defines seven layers (outer to inner): physical security, identity & access, perimeter, network, compute, application, and data.

#### Zero Trust

It is a security framework based on the premise "never trust, always verify".
There are 3 core principles in Zero Trust:

- **Verify explicitly:** Always authenticate and authorize based on all available data (users, endpoint, location, and risk).
- **Least privilege:** Grant only the needed privilege and only when it is needed.
- **Assume breach:** Act as if you are already compromised, segment, encrypt, and monitor everything.

Also there are 6 control areas:

- **Identity:** Every request needs to be authenticated and authorized.
- **Endpoint:** The device also needs to fulfill Zero Trust compliance requirements.
- **Network:** Network needs to be segmented and encrypted.
- **Infrastructure:** VMs, containers, and services must be hardened, monitored, and provided with just-in-time access.
- **App:** Apps must request only required permissions.
- **Data:** Data must be classified, labeled, and protected wherever it is.

#### Network Security Group

A NSG is a virtual firewall used to control inbound and outbound traffic to and from cloud resources like virtual machines or subnets. We define a set of rules that determine which traffic is allowed or denied.

#### Azure Firewall

Azure Firewall is a managed, cloud-based network security service that protects your Azure Virtual Network resources.
Azure Firewall has five key characteristics:

- **Stateful:** It monitors the complete state of network connections.
- **High Availability:** No extra configuration is required.
- **Scalability:** It scales automatically as your traffic grows.
- **FQDN filtering:** You can limit outbound traffic to specific domain names.
- **Centralized:** You can manage policies across multiple subscriptions and virtual networks.

#### DDOS Protection

In Azure, every resource is protected by a basic DDOS protection, but it is limited. There is also a Standard DDOS Plan, which can be linked to multiple VMs. The standard plan is an enhanced version of the basic, and includes metrics, ML, reports, rapid response, and cost protection.

#### Microsoft Entra Permissions Management

It helps organizations enforce least-privilege access, helps us inventory permissions, identify high risk or unused permissions, and apply on-demand elevation. It works across multiple clouds.

#### Workload Identities

It is an identity you assign to an application, service, script or container, so they can authenticate and access other services in a secure way.

#### Identity Governance

It is the process of managing and controlling user access across the organization.
This includes:

- **Lifecycle Workflows:** automate user changes (join, moves, or leave).
- **Entitlement Management:** request and assign access.
- **Access Reviews:** check if access is still needed.
- **PIM:** activate admin roles only when needed.

#### AD DS on-premises

Classic Active Directory that lives in on-prem servers, and uses GPOs, OUs, Kerberos, and LDAP.

#### Entra Domain Services

AD managed inside a VNet, without needing our own Domain Controllers. Useful with legacy apps that need Kerberos or LDAP in the cloud.

#### Entra Connect Sync

Synchronize users between on-prem AD DS and Entra ID. To do this it needs a server. Used to create hybrid identities.

#### Entra Cloud Sync

Lighter and more modern synchronization. Without a dedicated server.

#### Microsoft Entra External ID

Microsoft Entra External ID is used to manage access for users outside the organization.

#### B2B

B2B Collaboration allows external partners, suppliers, or guests to access company resources. External users are usually invited as guest users into the organization's tenant.

#### B2B Direct Connect

B2B Direct Connect allows direct collaboration between Microsoft Entra tenants. It uses a mutual trust relationship between organizations and is commonly used for Microsoft Teams shared channels.
Unlike B2B Collaboration, it does not require creating traditional guest users.

#### B2C

B2C is used for customers who need to sign in to customer-facing applications. Customer identities are managed in a separate external or customer tenant.
They can sign in with social IDs like Google accounts, Facebook accounts, and more, or they can use a local account.

#### Single Sign-On

Single Sign-On allows users to sign in once and access multiple applications without entering their credentials again.

#### Multi-Factor Authentication

Multi-Factor Authentication requires two or more verification factors. MFA adds an extra layer of security beyond passwords.

#### Conditional Access

Conditional Access uses policies to control access based on conditions such as identity signals, device compliance, location, or risk. Can be used to ensure that users only access applications from approved or compliant devices.

#### Role Assignment

A role assignment has three main elements:

- **Security principal:** Who
- **Role definition:** What
- **Scope:** Where

#### Security Principal

A security principal is the identity that receives access. It could be a user, group, service principal, or managed identity.

#### Role Definition

A role definition describes what actions are allowed.

- **Owner:** Full access, including managing access
- **Contributor:** Can manage resources, but cannot assign access
- **Reader:** Can view resources only
- **User Access Administrator:** Can manage user access

#### Scope

Scope defines where the permissions apply.
Scope levels:

- **Management Group**
- **Subscription**
- **Resource Group**
- **Resource**

#### Inheritance

Permissions assigned at a higher scope are inherited by lower scopes.

### 9. Governance and compliance

#### Resource Locks

Resource locks are used to protect Azure resources from accidental deletion or modification. They can be applied at different scopes (subscription, resource group, or resource) and are inherited by lower scopes.

- **CanNotDelete:** Users can modify the resource, but cannot delete it.
- **ReadOnly:** Users cannot modify or delete the resource.

Use a lock when you need to prevent deletion or modification of important resources, regardless of who has access to them.

#### Tags

Azure Tags are name-value pairs used to organize Azure resources and track costs by department, project, or environment.
Tags are not automatically inherited by default.
Azure Policy can be used to require, add, or inherit tags.

#### Azure Policy

Azure Policy is a governance service used to create, assign, and manage rules for Azure resources.
It helps ensure that resources follow organizational standards and compliance requirements.

Common uses:
- Restrict deployment of resources to specific regions
- Enforce naming conventions
- Limit VM sizes (e.g., only allow VMs with 4 or less vCPUs)
- Require specific tags on resources

#### Policy Assignment

A policy assignment applies a policy definition to a specific scope, and it is inherited.

#### Initiative

An initiative is a collection of policy definitions. Azure provides built-in initiatives, but organizations can also create custom initiatives.

#### Microsoft Purview

Microsoft Purview is used for data governance, data protection, data sharing, and compliance.
It helps organizations understand where their data is, what type of data they have, who can access it, and how it should be protected.
Microsoft Purview has free and enterprise capabilities.

#### Data Map

The Data Map is an inventory of data assets and metadata discovered through scans.

#### Unified Catalog

The Unified Catalog allows users to search, understand, and discover governed data.

#### Data Sharing

Data sharing allows users or teams to request and access data in a controlled and governed way.

#### Purview Policy

Purview policies help govern access and usage of data.
They can define who can access data, how access is approved, and under what conditions data can be used.

#### Classification and Labels

Classification identifies the type of data. Sensitivity labels classify data based on its sensitivity.

#### Data Estate Insights

Data Estate Insights provides analytics and reports about the organization's data estate.

#### Azure Blueprints

Azure Blueprints is used to create repeatable and governed Azure environments.

A blueprint can include artifacts such as:

- **Resource Groups**
- **ARM templates**
- **RBAC role assignments**
- **Azure Policy assignments**
- **Resource Locks**

#### Resource Locks in Blueprints

Blueprint assignments can apply locking options such as:

- **Do not lock**
- **Do not delete**
- **Read only**

#### Cloud Adoption Framework

The Microsoft Cloud Adoption Framework is a set of guidance and best practices to help organizations adopt Azure successfully.

- **Strategy:** Define business goals and cloud outcomes
- **Plan:** Create a cloud adoption plan
- **Ready:** Prepare the Azure environment and landing zones
- **Adopt:** Migrate, modernize, or build workloads
- **Govern:** Apply governance, compliance, cost control, and policies
- **Manage:** Operate, monitor, and maintain cloud resources
- **Secure:** Protect cloud workloads, identities, data, and resources

#### Microsoft Privacy Statement

Describes which personal data Microsoft processes, how, and for what purposes.

#### Online Services Terms (OST)

It is the legal agreement between Microsoft and the customer. It outlines the various obligations around processing and security of data.

#### Data Protection Addendum (DPA)

### 10. Cost management and SLAs

#### Factors That Affect Azure Costs

The cost of Azure resources depends on what you use, how much you use, where you deploy it, and which service options you choose.

- **Resource Type:** Different Azure services have different pricing models, such as Virtual Machines, Storage Accounts, Azure SQL Database, or Azure Functions.
- **SKU:** Defines the service option, size, or performance level. Higher SKUs usually cost more.
- **Tier:** Defines the service level, such as Free, Basic, Standard, or Premium. Higher tiers usually include more features and higher cost.
- **Location:** Azure prices can vary depending on the region where the resource is deployed.
- **Meters:** Units used to measure Azure consumption, such as VM hours, GB stored, requests, transactions, executions, or licenses.
- **Running vs Existing:** Some resources cost money while running, while others can cost money just by existing or storing data.
- **Instances:** More instances usually increase cost, such as running multiple VMs or scaling an App Service.
- **Serverless Work:** Serverless services often charge based on executions, requests, or resource consumption.
- **Storage:** Costs can depend on the amount of data used, provisioned capacity, access tier, transactions, and data retrieval.
- **Interactions:** Some services charge for operations or transactions, such as read operations, write operations, API calls, or data retrieval.
- **Licensing:** Some services include licensing costs, such as Windows Server, SQL Server, or premium Microsoft licenses.

#### Azure Reservations

Azure Reservations help reduce costs by committing to use certain Azure resources for 1 year or 3 years.
They are useful for **predictable, long-term workloads** (production VMs with consistent usage throughout the year).
Reservations can save up to 72% compared to pay-as-you-go pricing.

#### Spot VMs

Azure Spot VMs allow you to use unused Azure compute capacity at a lower cost.
They are cheaper than regular virtual machines, but Azure can evict them when capacity is needed.

#### Deallocate vs Stop

For VMs that are only used during certain periods (e.g., 9:00-17:00):
- **Deallocate** the VM when not needed → stops compute charges, preserves disks and data.
- Just stopping the VM does not always stop compute charges depending on configuration.

#### Azure Cost Management

Azure Cost Management helps monitor, analyze, and control Azure spending.
It allows organizations to understand where costs come from and optimize cloud expenses.

#### Service Level Agreements

A Service Level Agreement (SLA) describes the expected availability and uptime of an Azure service, usually expressed as a percentage (99.9%, 99.95%, 99.99%). Free services do not include a financial SLA.

#### Composite SLA

Used when an application depends on multiple services. The total SLA is calculated by multiplying the SLAs of each service, which means the combined SLA is always lower than the individual ones.

#### Azure Service Lifecycle

The Azure Service Lifecycle describes the stages an Azure service goes through before becoming fully available. The typical path is: Internal → Private Preview → Public Preview → General Availability (GA).
