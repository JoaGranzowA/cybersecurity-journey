### Cloud computing fundamentals

#### What is cloud computing?

Cloud computing is on-demand delivery of computing services, instead of having on-premise servers, you rent these resources like storage, databases, and software.

#### Shared responsibility model

Shared responsibility model works in the way that the client and the cloud service provider share responsibility, it defines which security tasks are handled by the cloud provider and which are handled by the client.

#### Cloud deployment models: public, private, hybrid, multi-cloud

**Public Cloud:** In public cloud services are delivered over the public internet and shared over multiple organizations.

**Private Cloud:** A private cloud is used exclusively by a single organization. It can be hosted either on-premises or by a third-party provider, but its resources are not shared with others.

**Hybrid Cloud:** Hybrid cloud uses both public and private, usually they store the sensitive data on-premise and the other type of data in the public cloud.

**Multi-Cloud:** The use of 2 or more different public cloud providers.

#### Cloud service types: IaaS, PaaS, SaaS

**IaaS:** In infrastructure as a service you rent the resources, like the virtual servers, the storage, and networks. You are responsible for installing and managing the OS and the tools.

**PaaS:** In platform as a service you pay for a platform for developers to build, test, and deploy apps without worrying about server management.

**SaaS:** In software as a service you pay for the final product, the app is ready to use. You simply log in and use it.

#### CapEx vs OpEx

**CapEx:** Capital expenditure means that you spent money building infrastructure, maintain, or improve before you can use it, CapEx requires large upfront investment + long depreciation.

**OpEx:** In operational expenditure you pay as you use the resources, you don't have to maintain or improve the datacenters. You can stop, start, or scale the resources based on demand, OpEx spreads cost over time.

#### Consumption-based pricing

Consumption-based pricing is applied on OpEx, you pay as you go, no upfront commitment, and you can stop, scale up or scale down services based on demand.

### Azure core architecture

#### Regions, availability zones, region pairs

A region is a geographic location, where a provider has multiple data centers. An availability zone is a physically separate datacenter within an Azure region, each with independent power, cooling, and networking. Availability zones exist so that if one zone fails, others continue operating.

A region pair consist of 2 regions within the same geography, these help to save data in case of a disaster recovery. Region pairs are usually at least 300 miles apart and Azure platform updates are rolled out sequentially to minimize risk of simultaneous failure.

#### Resources, resource groups, subscriptions

**Resources:** Are instances of a service that we can create and manage.

**Resource groups:** Logical containers used to group and manage resources.

**Subscriptions:** Billing and access boundaries that contain resource groups and resources. They help separate environments, projects, departments, or cost centers.


#### High availability, scalability, reliability, predictability

**High availability:** The ability of a system to remain accessible and operational with minimal downtime, even during failures.

**Scalability:** The ability of a system to handle increased demand by adding or adjusting resources.

**Reliability:** The ability of a system to consistently perform its intended function without failure over time.

**Predictability:** The ability to ensure consistent performance and cost, so the system behaves as expected under different conditions.

#### Vertical vs horizontal scaling

**Vertical scaling:** Increasing the capacity of a single resource by adding more power.

**Horizontal scaling:** Increasing capacity by adding more instances of a resource and distributing the load.

#### Azure AD/Entra ID

Entra ID is a cloud-based identity and access management service that manages users and groups, authenticates sign-ins, and control access to resources using roles.
Azure subscriptions trust only one Entra ID tenant.

#### Management Groups

Management groups help to apply RBAC, policies, budget, and manage multiple subscriptions from a single point.
Management groups have a hierarchical structure, where configurations are inherited to the child management groups, subscriptions, and resources. A subscription can belong to only one management group at a time.

#### Azure Resource Manager(ARM)

ARM is the deployment and management service for Azure, it creates, updates, and deletes resources. ARM not only does that, it also applies authorization and policies. It deploys resources using templates(JSON/Bicep). One key thing to keep in mind is that all tools interact with ARM, so you cannot bypass it.

#### Azure Arc

Azure Arc extends management and governance to resources outside Azure, which means we can manage on-premises servers, virtual machines in other clouds, Kubernetes clusters, etc.
With Azure Arc, we can apply policies, RBAC, security features, and more to those resources.

#### Resources Required for Virtual Machines

A virtual machine is not a single resource; it depends on multiple components, including compute, storage, networking, and security.
Choosing the right VM size (CPU and memory) is important for performance and cost.

#### Virtual Machine Scale Sets

Virtual Machine Scale Sets (VMSS) allow you to deploy and manage multiple identical VMs and scale them automatically based on demand.

#### Azure Container Instances

Azure Container Instances (ACI) allow you to run containers without managing virtual machines, providing a simple and fast way to deploy applications. You define CPU and memory at deployment time.

#### Azure Kubernetes Service

Azure Kubernetes Service (AKS) is a managed Kubernetes service that orchestrates and scales containerized applications across multiple nodes.

#### Azure App Service

Azure App Service is a PaaS offering that allows you to build and host web applications without managing infrastructure.

#### Azure Serverless

Azure Serverless allows running code without managing infrastructure using an event-driven model. Azure Functions executes code in response to events, while Logic Apps automate workflows and integrations.

#### Azure Virtual Desktop

Azure Virtual Desktop (AVD) allows users to access virtual desktops and applications hosted in Azure from any device.

#### Azure Virtual Networks

Azure virtual networks are used to create private networks in Azure. A VNet is limited to a single region and cannot span across regions.
VNets use private IP address ranges defined by RFC1918.

#### Virtual Networks Subnets

Subnets are created within a VNet by dividing the IP address range into smaller segments, so each subnet must have a defined IP range.

#### VNet Peering

VNet peering allows communication between Vnets using private IP addresses.
When using multiple VNets, their IP address ranges must not overlap.

#### Site-to-Site VPN

Site-to-Site VPN is used to connect an on-premises network to an Azure virtual network over the internet. It creates an encrypted tunnel between the networks and allows them to communicate using private IPs.
It requires a VPN gateway in Azure and a compatible VPN device on-premises.
Site-to-Site VPNs can be configured as route-based or policy-based.

#### Policy-based VPNs

Policy-based VPNs use policies to define which traffic is encrypted and sent through the tunnel. They are more limited and typically used for compatibility with older VPN devices.

#### Route-based VPN

Route-based VPNs use routing tables to direct traffic through the VPN tunnel and support dynamic routing. They are more flexible and recommended for most scenarios.
Route-based VPNs support both Site-to-Site (S2S) and Point-to-Site (P2S) connections.

#### Point-to-Site

Point-to-Site (P2S) allows individual clients (VMs, laptops, etc) to connect securely to an Azure Virtual Network.

#### Active-Passive configuration

Active-Passive (A-P) configuration uses one active VPN gateway and one standby instance for failover.

#### Active-Active configuration

Active-Active (A-A) configuration uses two active VPN gateway instances for higher availability and better performance.

#### Azure ExpressRoute

ExpressRoute provides a private and dedicated connection between on-premises infrastructure and Azure without using the public internet.
Traffic flows through a carrier over a private network to Microsoft routers. It offers high reliability, lower latency, and more consistent performance compared to VPN connections.

#### Private Peering

Private Peering is used to connect on-premises networks to Azure Virtual Networks using private IPs. It uses ExpressRoute to keep the connection private.

#### Microsoft Peering

Microsoft Peering is used to access Microsoft public services such as Microsoft 365 and Azure PaaS services. It uses public IP addresses, but traffic still travels over the private ExpressRoute connection.

#### Service Endpoints

Service endpoints allow Virtual Networks to connect to Azure services using the Azure backbone network. They extend the VNet identity to Azure services such as Azure Storage and Azure SQL Database. 
Service Endpoints use public IP addresses of the Azure service but restrict access to specific VNets and subnets.

#### Private Endpoints

Private Endpoints provide a private connection from a Virtual Network to Azure services using a private IP address. They assign a private IP from the VNet to the Azure service, making it accessible as if it were inside the VNet. Also we can connnect on-premises networks through a VPN or ExpressRoute.

#### Public Endpoints 

It is an address that we can communicate to from the internet(Public) or from services in a virtual network. It is internet routable, i can get to it from anywhere, but i still need to authenticate depending on the service.
We could use a service level firewall to allow communication only from specific public IPs or VNets. It is the default for most PaaS services.

#### Storage Account

A storage account is a service used to store different types of data, such as blobs, tables, queues, and files.
Storage accounts have two performance options: Standard, and Premium.
Standard performance uses GPv2(General Purpose v2).
On the other side, Premium performance has three options, Block blob, Page blob, and File Shares.

#### Locally Redundant Storage

In LRS there are three copies of the data, but they are in the same datacenter within the region. If something happens to that datacenter, our data may be not available.

#### Zone Redundant Storage

ZRS also keeps three copies of the data, but they are stored in different zones. In this case we still have access to the data if one availability zone goes down.

#### Geo-Redundant Storage

GRS keeps six copies of the data: three in the primary region and three in a secondary region, which is the paired region. This protects against a full regional failure.
The replication to the secondary region is asynchronous, so if the primary region goes down suddenly, we could lose the last few seconds of writes that were not yet replicated.

#### Geo-Zone-Redundant Storage

GZRS is a combination of GRS and ZRS. It keeps six copies of the data. There are three copies in the primary region, but in different availability zones, and the other three are stored in a secondary region.

#### Standard vs Premium

Standard has LRS, ZRS, GRS, and GZRS.
Premium has only LRS and ZRS.
This means Premium does not support geo-redundancy, so if a workload needs both high performance and protection against regional failures, the redundancy must be handled at the application level.

#### GRS and GRZS Read Access

The Read Access option allows us to get a second endpoint in the secondary region to read the data.

#### Binary Large Object(Blob)

A Blob is basically any type of data we want to store, such as images, videos, documents, backups, logs, or files.

#### Block Blob

A Block Blob is used to store files such as images, videos, documents, backups, and logs. It stores the data in blocks and is the most common type of Blob.
Block blobs use a flat namespace, which means there is no real folder hierarchy.

#### ADLS Gen2

ADLS Gen2 is built on top of Blob Storage and is used to store large amounts of data for analytics and big data workloads. It also supports a hierarchical namespace.

#### Page Blob

A Page Blob is used to store virtual hard disks for Azure Virtual Machines. It is optimized for frequent read and write operations.

#### Append Blob

An Append Blob is used to append data to the end of a blob. It is commonly used for logs.

#### Access Tiers

Access tiers define how frequently the data is accessed. Hot is for frequently accessed data, Cool is for less frequently accessed data, Cold is for rarely accessed data, and Archive is for long-term storage.

#### Standard HDD

Standard HDD is the cheapest disk option. It uses traditional hard disk drives and is used for workloads that do not need high performance.
It is commonly used for backups, test environments, or infrequently accessed workloads.

#### Standard SSD

Standard SSD provides better performance than Standard HDD and is used for light workloads, web servers, dev/test environments, and small applications.
It also supports burst performance, which allows the disk to temporarily increase its performance when there are short peaks of activity.

#### Premium SSD

Premium SSD is used for production workloads that need high performance and low latency. It supports burst performance, which is useful when the workload has temporary peaks of activity. It also supports performance tiers, which allow us to increase the disk performance without increasing the disk size.

#### Premium SSD v2

Premium SSD v2 provides higher performance and more flexibility than Premium SSD. It allows us to configure the disk size, IOPS, and throughput independently.
Premium SSD v2 is dynamic because we can adjust IOPS and throughput depending on the workload.

#### Ultra Disk

Ultra Disk is the highest performance disk option in Azure. It is used for very demanding workloads, such as large databases, SAP HANA, and applications that require very high performance. It allows us to configure IOPS and throughput independently.
Ultra Disk is dynamic because we can change its performance settings without restarting the VM.

#### Azure Files

Azure Files is a fully managed file share service in Azure. It allows us to create shared folders in the cloud that can be accessed by multiple users, applications, or virtual machines.

#### Azure File Sync

Azure File Sync is a service that allows us to synchronize an on-premises file server with an Azure file share.
It lets users keep using a local file server, while Azure keeps a synchronized copy in the cloud.

#### Queue Storage

Queue Storage is used to store messages that can be processed later. It works with FIFO, which means First In, First Out. This means that the first message added to the queue is usually the first message to be processed.
Queue Storage is commonly used to communicate between different parts of an application.

#### Table Storage

Table Storage is used to store structured NoSQL data. It stores data as key-value pairs, which means each item has a key that identifies it and a value that contains the information.
It is useful for simple and fast data storage without using a relational SQL database.

#### Static Website

Static Website allows us to host static web content directly from a storage account.
